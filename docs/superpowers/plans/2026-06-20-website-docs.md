# Website Docs Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a polished Astro + Starlight GitHub Pages website for `telos-agent`, including a custom homepage, initial docs, copied Rust API docs, and automated Pages deployment.

**Architecture:** The website is isolated in `site/` and is not part of the Rust workspace. Astro renders the custom homepage and Starlight documentation; a root script builds the static site, runs rustdoc, and copies `target/doc` into `site/dist/api/rust/` before deployment.

**Tech Stack:** Astro 4.16.18, Starlight 0.29.0, TypeScript 5.9.3, GitHub Pages Actions, Rust rustdoc.

---

## File Structure

- Create `site/package.json`: Node package, Astro scripts, pinned compatible dependencies.
- Create `site/astro.config.mjs`: Astro/Starlight configuration with GitHub Pages `site` and `base`.
- Create `site/tsconfig.json`: Astro TypeScript config.
- Create `site/src/content/config.ts`: Starlight docs collection schema for Astro 4.
- Create `site/src/pages/index.astro`: custom homepage.
- Create `site/src/styles/global.css`: homepage and Starlight theme styling.
- Create `site/src/content/docs/*.mdx`: initial docs pages.
- Create `scripts/build-site.sh`: single command to build docs and copy rustdoc into site output.
- Create `.github/workflows/site.yml`: GitHub Pages build and deploy workflow.
- Modify `.gitignore`: ignore `site/node_modules/`, `site/dist/`, and `site/.astro/`.
- Create `site/package-lock.json`: generated by `npm install` inside `site/`.

## Implementation Notes

- Use Astro 4 because this machine has Node `v18.19.1`; latest Astro 6 requires Node `>=22.12.0`, and Astro 5 requires newer Node 18 patch releases.
- Use `import.meta.env.BASE_URL` for homepage links so local dev and GitHub Pages base paths both work.
- Generated rustdoc is copied after `astro build`, because Astro clears `site/dist` during builds.
- Do not commit generated `site/dist`, `target/doc`, or `.superpowers/`.

### Task 4: Add Rust API Copy Build Script

**Files:**

- [ ] **Step 1: Create `scripts/build-site.sh`**

```bash
#!/usr/bin/env bash
set -euo pipefail

ROOT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
SITE_DIR="$ROOT_DIR/site"
RUSTDOC_OUT="$ROOT_DIR/target/doc"
API_OUT="$SITE_DIR/dist/api/rust"

cd "$ROOT_DIR"

export RUSTDOCFLAGS="${RUSTDOCFLAGS:--D warnings}"
./scripts/generate-core-api-docs.sh

cd "$SITE_DIR"
npm run build

rm -rf "$API_OUT"
mkdir -p "$API_OUT"
cp -R "$RUSTDOC_OUT"/. "$API_OUT"/

test -f "$API_OUT/telos_agent/index.html"
echo "Site built with Rust API docs at $API_OUT"
```

### Task 5: Add GitHub Pages Workflow

**Files:**

- [ ] **Step 1: Create `.github/workflows/site.yml`**

```yaml
name: Site

on:
  push:
    branches: ["main"]
  pull_request:
    branches: ["main"]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    name: Build site
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
          cache-dependency-path: site/package-lock.json

      - uses: dtolnay/rust-toolchain@stable

      - uses: actions/cache@v4
        with:
          path: |
            ~/.cargo/registry/index/
            ~/.cargo/registry/cache/
            ~/.cargo/git/db/
            target/
          key: ${{ runner.os }}-cargo-site-${{ hashFiles('Cargo.lock') }}
          restore-keys: ${{ runner.os }}-cargo-site-

      - name: Install site dependencies
        working-directory: site
        run: npm ci

      - name: Build site and Rust API docs
        run: ./scripts/build-site.sh

      - name: Upload Pages artifact
        if: github.event_name == 'push' && github.ref == 'refs/heads/main'
        uses: actions/upload-pages-artifact@v3
        with:
          path: site/dist

  deploy:
    name: Deploy site
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```
